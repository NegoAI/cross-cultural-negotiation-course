# Final Submission

What your group hands in. Place these files in this folder before the deadline.

## 1. The deployed agent

A file called `agent-link.md` containing:

- The public link to your deployed Claude Project or ChatGPT Project
- The platform you used (Claude Project / ChatGPT Project) and a one-line note on why your group picked it
- Any access notes I will need to use the agent (login, permissions, how to start a conversation)
- The version date — when the System Instructions you are submitting were last deployed

## 2. The artefacts you produced

Three files (or folders), in whatever form your group wrote them:

- **The Knowledge Base** — the document attached to the deployed agent
- **The PRD** — the Product Requirements Document you built in Step 3
- **The System Instructions** — the operational text pasted into your Claude Project or ChatGPT Project

All four submission pieces — these three plus the evaluation report below — are in Markdown (`.md`). See the *File format* section in the project README for the reason. What I grade is the rigour of the thinking; the format requirement is technical.

## 3. The Evaluation Report

A single file covering Part A (self-evaluation) and Part B (real-case test with one round of iteration), per `5-evaluation.md`. Include the input you gave the agent, the agent's first output, the SI changes you applied, and the agent's second output — all in full.

## 4. Group manifest

A short file — `group.md` — naming the group members and a one-line note on who led which step. I am not policing equal contribution, but I want to know who did what.

## What I check first

1. The agent link works and the agent boots
2. Your real-case input runs through the deployed agent and produces an output I can read
3. Your evaluation honestly shows what the agent does well and what it does poorly

If those three pass, I read the rest. If any of them fails, the rest of the submission is read with that context.

## What I will not accept

- A submission with no working agent
- An evaluation tested only on the cultural axis your group knows best — the rubric tests generalisation
- A KB with citations you have not personally verified on Google Scholar or in a library catalogue
- A KB that contains behavioural directives, or System Instructions that contain uncited domain facts
